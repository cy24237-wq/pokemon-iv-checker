# Pokemon IV Checker
ウェブURL:https://cy24237-wq.github.io/pokemon-iv-checker/
## 概要
趣味で開発したポケモンの個体値チェッカーです。
人気ゲーム「ポケモン スカーレット・バイオレット」では、各ポケモンに種族値と呼ばれる能力値が設定されています。種族値はポケモンごとに固定されていますが、それとは別に個体ごとに個体値が存在し、同じポケモンであっても能力値に差が生じます。

対戦環境では個体値が重要な要素となるため、育成を行う際には個体値の確認や評価が欠かせません。しかし、ゲーム内だけでは評価が分かりにくく、複数の情報を参照しながら判断する必要があります。

そこで、ポケモン名や能力値を入力するだけで個体値を評価し、育成に適した個体かどうかを分かりやすく確認できるWebアプリを開発しました。本アプリでは、ポケモンの種族値データを管理し、ユーザーが直感的に利用できる画面設計を目指しました。


## 制作背景

ポケモン対戦では、ポケモンごとに定められた種族値に加え、個体ごとに異なる個体値を考慮しながら育成方針を決定する必要があります。しかし、既存のWebアプリには種族値を調べるツールや個体値を確認するツールは存在するものの、「その個体値がそのポケモンに適しているか」を総合的に判断できるツールはほとんどありません。

私自身、ポケモン対戦を始めた当初は、手に入れたポケモンの個体値が優秀なのか、それとも育成に向いていないのか判断することができませんでした。同様の課題を抱える初心者プレイヤーも多いと考えています。

そこで、ポケモンの種族値や特徴と個体値を組み合わせて評価し、育成に適した個体かどうかを一つの画面で分かりやすく確認できるWebアプリを開発しました。

## 技術選定の理由

本アプリは、ユーザーがブラウザ上で手軽に利用できることを重視して開発しました。そのため、特別な環境構築が不要なHTML・CSS・JavaScriptを採用しています。

また、ポケモンの種族値データを手入力で管理するのではなく、PokeAPIから自動取得する仕組みを構築しました。取得したデータはNode.jsを用いてJSON形式へ変換し、フロントエンドから利用することで保守性と拡張性を向上させています。

開発したアプリはGitHubでソースコードを管理し、GitHub Pagesを利用して公開することで、実際に動作する成果物として閲覧できるようにしています。

## 前提知識-個体値とは

ポケモンシリーズでは、各ポケモンに「種族値」と「個体値」という2種類の能力値が存在します。

種族値はポケモンの種類ごとに決められた固定値であり、例えばピカチュウであれば全ての個体が同じ種族値を持ちます。一方、個体値はポケモンごとに異なる値であり、HP・攻撃・防御・特攻・特防・素早さの6項目についてそれぞれ0〜31の範囲で設定されています。

そのため、同じポケモンであっても個体値の違いによって最終的な能力値が変化します。特に対戦では個体値が勝敗に影響を与えるため、プレイヤーは個体値を考慮して育成を行います。

また、ポケモンによって重視される能力は異なります。例えば、物理攻撃を主体とするポケモンでは攻撃の個体値が重要視される一方で、特殊攻撃を使用しないポケモンでは特攻の個体値が低い方が有利になる場合もあります。そのため、「個体値が高い＝必ず優秀」とは限らず、ポケモンの特徴に応じた評価が必要になります。

## 評価アルゴリズム

本アプリでは、全てのポケモンを同じ基準で評価するのではなく、ポケモンの役割に応じて個体値の重要度を変化させています。

一般的な個体値チェッカーでは、全ての能力値が高い個体が高評価となります。しかし実際の対戦では、ポケモンの役割によって重要な能力値は異なります。

そのため、本アプリでは以下の5つの役割を定義し、能力ごとに重み付けを行っています。

| 役割                | 評価方針                  |
| ----------------- | --------------------- |
| Physical Attacker | 攻撃・素早さを重視             |
| Special Attacker  | 特攻・素早さを重視             |
| Slow Tank         | HP・防御・特防を重視し、素早さを低評価  |
| Trick Room        | トリックルーム運用を想定し、素早さを低評価 |
| Balanced          | 全能力を均等評価              |

例えば物理アタッカーでは攻撃と素早さに高い倍率を設定し、特殊アタッカーでは特攻と素早さを重視しています。

また、耐久型やトリックルーム型では素早さが必ずしも高い方が有利ではないため、素早さの評価倍率を低く設定しています。

この仕組みにより、単純な高個体値のポケモンではなく、そのポケモンの役割に適した個体を高く評価できるようにしています。
