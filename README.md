# Pokemon IV Checker
ウェブURL:https://cy24237-wq.github.io/pokemon-iv-checker/
## 概要
人気ゲーム「ポケモン スカーレット・バイオレット」では、各ポケモンに種族値と呼ばれる能力値が設定されています。種族値はポケモンごとに固定されていますが、それとは別に個体ごとに個体値が存在し、同じポケモンであっても能力値に差が生じます。

対戦環境では個体値が重要な要素となるため、育成を行う際には個体値の確認や評価が欠かせません。しかし、ゲーム内だけでは評価が分かりにくく、複数の情報を参照しながら判断する必要があります。

そこで、ポケモン名や能力値を入力するだけで個体値を評価し、育成に適した個体かどうかを分かりやすく確認できるWebアプリを開発しました。本アプリでは、ポケモンの種族値データを管理し、ユーザーが直感的に利用できる画面設計を目指しました。

## 制作背景
ポケモン対戦では、個体値や種族値を確認しながら育成方針を決める必要があります。  
このWebアプリの開発当初の私をはじめ、多くのポケモン初心者はゲーム内で手に入れたポケモンの個体値がそのポケモンにとっていいものなのか判断がつかないです。
個体値があっているかは複数の情報を別々に確認するのは手間がかかるため、必要な情報を1つの画面で確認できるツールを作成したいと思いました。

## 使用技術

| 技術         |選定理由                                         
| ------------ | -------------------------------------------- 
| HTML         | ユーザーがポケモン情報を入力・閲覧できる画面を構築するため                
| CSS          | 検索結果や評価情報を見やすく表示し、直感的に利用できるUIを実現するため         
| JavaScript   | ポケモン検索機能や個体値評価機能などのフロントエンド処理を実装するため          
| Node.js      | PokeAPIから大量のポケモンデータを取得し、JSONファイルとして生成・管理するため 
| PokeAPI      | ポケモンの種族値やタイプなどの公式データを取得するため                  
| JSON         | ポケモンデータを構造化して管理し、高速な検索・表示を実現するため             
| GitHub       | ソースコードの管理およびバージョン管理を行うため                     
| GitHub Pages | 開発したWebアプリを公開し、誰でも利用できるようにするため               

## 技術選定の理由

本アプリは、ユーザーがブラウザ上で手軽に利用できることを重視して開発しました。そのため、特別な環境構築が不要なHTML・CSS・JavaScriptを採用しています。

また、ポケモンの種族値データを手入力で管理するのではなく、PokeAPIから自動取得する仕組みを構築しました。取得したデータはNode.jsを用いてJSON形式へ変換し、フロントエンドから利用することで保守性と拡張性を向上させています。

開発したアプリはGitHubでソースコードを管理し、GitHub Pagesを利用して公開することで、実際に動作する成果物として閲覧できるようにしています。

## 主な機能
- ポケモン名検索
- 個体値評価
- 種族値データ表示
- JSONデータを用いたポケモン情報管理
- PokeAPIからのデータ取得

## データ構造

ポケモンデータはJSON形式で管理しています。

```json
{
  "name": "Pikachu",
  "dexNumber": 25,
  "types": ["Electric"],
  "stats": {
    "hp": 35,
    "attack": 55,
    "defense": 40,
    "specialAttack": 50,
    "specialDefense": 50,
    "speed": 90
  }
}
```
PokeAPIから取得したデータをJavaScriptで加工し、アプリケーション内で利用しやすい形式に変換しています。
